DoubleMap - Usability Evaluation

We conducted usability evaluation of DoubleMap with the live version of web and mobile application. Android base smartphone and iPhone were used to test the android and ios version of the application and Google chrome for testing web based app. For usability evaluation we followed various methods like cognitive walkthrough, contextual inquiry, interviews, heuristic evaluation to collect data from for the evaluation


Research and Analysis

Competetive Analysis

Transloc Rider: It is a similar app that provides real time bus updates and announcements. Many transit-systems and university campuses use it

RideCell: It is a company that provides ride / car sharing applications. It also provides services for fixed routes fleet automation. Works like Transloc and DoubleMap.


Overall DoubleMap stands out with most features on paper as compared to competition. Thorough usability tests and evaluations we will analyze how these features perform.

Contextual inquiry

Contextual Inquiry is a part of user experience research process in which we collaborate with the participant during the session we are observing and learning about how they use the application and perform the assigned tasks. The core tasks for our application is
  • Find route to a particular stop
  • Locate your own location
  • Add a stop to favorites
  • Get push notifications for a stop

Problems

  • New users took time to understand the application
  • iOS users took more time to perform task which asked them to get notification for a bus
  • In Android application, on setting the alert they were unable to delete it

Heuristic Evaluation

Process:The 10 heuristic evaluation usability principles by renowned usability expert Jakob Nielsen was used to produce the problems/good aspects about Double Map app. Each problem/feature was recorded in a Usability Aspect Report(UAR) The Usability Aspect Report was produced using the following steps:
  • Each member used the DoubleMap app (mobile based and browser based) several times and tried to accomplish few tasks.
  • Every action performed by the team members was compared against 10 heuristic principles mentioned above
  • Problems/Good aspects identified in actions performed were recorded with evidence (screenshots).
  • If the identified issue is a problem, severity rating was given to the problem based on Nielson's ratings.
  • Possible solutions/tradeoffs were provided for every problem identified

Problems

  • Mobile Application did not have any form of help, tutorial or instructional overlay
  • Search feature used for finding locations is case sensitive
  • Alerts functionality is inconsistent across transit systems in mobile application
  • GPS icon does not perform any action if GPS is not active

Recommendations

System should prompt the user to enable GPS if it is disabled
